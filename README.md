# appportal-leyu-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面主要作为静态内容展示，不绑定任何特定域名或网站。

## 项目简介

这是一个通用型的 HTML 页面存档仓库，旨在以结构化方式存放并发布各类独立页面。仓库内容以静态 HTML 文件为主，方便直接访问或嵌入其他环境。

## 目录结构

```
.
├── pages/          # 存放各独立 HTML 页面文件
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件代表一个独立页面，命名清晰，便于识别。
- `assets/`：存放多个页面可能共用的样式、脚本或媒体文件。
- 根目录下可放置索引或引导页面，便于浏览。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，可脱离当前仓库独立运行。
- 页面内容以归档和展示为主，不涉及动态交互或后端服务。
- 文件命名应反映页面主题或用途，避免歧义。

## 使用方式

1. 直接打开 `index.html` 或 `pages/` 下的任意 HTML 文件即可查看页面内容。
2. 可通过 GitHub Pages 或其他静态托管服务发布整个仓库，获得可公开访问的链接。
3. 如需添加新页面，请在 `pages/` 下创建新文件，并确保引用资源路径正确。
